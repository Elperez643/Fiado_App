# Secrets And Environments

No real secret belongs in Git.

Do not version:

- `.env` or `.env.*` except `.env.example`
- `appsettings.Development.json`
- `appsettings.Local.json`
- `appsettings.StagingLocal.json`
- `appsettings.*.local.json`
- `secrets.json` or `*.secrets.json`
- signing keys, certificates, service account files, or private keys
- local SQLite databases

Local backend configuration:

- Prefer .NET user secrets for developer-only values.
- Use environment variables such as `ConnectionStrings__FiadoDb` for local overrides.
- Keep templates with placeholders only.
- Do not paste production credentials into Flutter, markdown, tests, logs, or scripts.

Future Azure configuration:

- Azure Database for PostgreSQL Flexible Server will replace SQL Server in a separate migration project.
- Azure Storage Account and Blob Storage will store product images, logos, receipt PDFs, campaign resources, and exported reports.
- Azure Front Door will provide CDN, WAF, and perimeter routing.
- Flutter must not contain Azure secrets.
- Prefer Managed Identity, server-side tokens, or short-lived scoped access generated by the backend.

If a secret is found:

1. Stop staging the file.
2. Add or confirm the ignore rule.
3. Create a safe template.
4. Rotate the exposed secret if it ever left the local machine or entered history.
